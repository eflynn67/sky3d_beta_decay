subroutine, private grids::init_coord (name, nv, dv, v, der1v, der2v, cdmpv)
 In this subroutine the defining information for a grid direction (generically called v, which can be replaced by x, y, or z) in the form of the number of points nv and the spacing dv is used to generate the associated data. The arrays of coordinate values, derivative and damping matrices are allocated. Since all the quantities that are later used in the code are passed as arguments, this subroutine can handle all three directions in a unified way.
